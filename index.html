---

## EEW Architecture

### Pipeline
Detect P-wave -> Grouping -> Location -> Magnitude -> Intensity

### ML for Intensity
- Use **3 sec P-wave**, ~**15 sec leading time**

---

## Large Earthquake Model (LEM)

### SeisWav2Vec 2.0
- Adapted from Wav2Vec 2.0 (speech), contrastive learning
- Self-supervised: extracts features from unlabeled data

### Components
1. Feature Encoder: waveform -> embeddings
2. Context Encoder: 12-layer Transformer
3. Masking & Quantization: masked prediction

### Workflow
Pre-training (unlabeled) -> Fine-tuning (CNN Decoder) -> Inference

---

## Hermes Multi-Agent System

| Agent | Role | Responsibilities |
|-------|------|-----------------|
| my_agent | EEW Manager | Cron, anomaly alerts, daily summary |
| secondary_agent | Integration Engineer | Hermes forwarding, task dispatch |
| seismo_agent | Seismology Assistant | .rep parsing, latency, plots |
| seismo | Researcher | Modeling, experiments, trends |

### Automated Workflow
Event trigger (.rep) -> Data dispatch -> Plotting -> Modeling -> Report

---

## AI Workflow Evolution

| Period | Phase | Key Feature |
|--------|-------|-------------|
| 2022-2024 | Chat Era | ChatGPT, AI as assistant |
| 2024-2025 | IDE Integration | Cursor, Windsurf, Copilot Agent |

---

## Conclusion
- SSL enables feature extraction from unlabeled data
- LEM brings new era for seismic monitoring
- Multi-agent collaboration reshaping EEW workflows

03 Taiwan EEW Evolution

# Evolution of Taiwan Earthquake Early Warning System

- Date: 2026-06-12  |  Event: Chiayi County Disaster Prevention Training
- Speaker: Da-Yi Chen  |  Organization: CWA Seismological Center

---

## Taiwan Seismic Environment

- Philippine Sea vs. Eurasia: **7-8 cm/yr** convergence
- **7 major disasters** since 1900 (>100 deaths each)
- Daily ~100 quakes, annual ~100 felt, every 30-40 yr major disaster

### Recent Disasters (10yr)
- 2016 M6.6 Meinong: 1069 casualties
- 2018 M6.2 Hualien: 308 casualties
- 2024 M7.1 Hualien: 621 casualties
- 2025 M6.4 Dapu: 1810 houses damaged

---

## Warning Principle

- P-wave: fast, small amplitude | S-wave: slow, destructive
- Estimate magnitude from P-wave before S-wave arrives

### Alert Conditions
- M >= 4.5 with intensity >= 4, or M >= 6 with intensity >= 3

### Channels
PWS (cell broadcast), TV, internet, App, LINE, website

---

## Race Against Seismic Waves

| Period | Warning Time | Blind Zone |
|--------|-------------|------------|
| 30 yr ago | **102 sec** | - |
| Current | **7 sec** | **25 km** (50% area reduction) |

---

## Technical Limitations

### False Alarms
- 2022/09/18 false PWS in Yunlin-Chiayi-Tainan
- Double earthquakes cause misjudgment

### Missed Events
- Two events **4 sec apart**; system processed only first

### Deep Earthquakes
- >100km depth may underestimate surface intensity

### Tsunami

| Trench | Location |
|--------|----------|
| Ryukyu | East Taiwan |
| Yap | Southeast Taiwan |
| Manila | South Taiwan |

- **6 warning zones**, **4 wave-height levels**

#### Case: 2024/04/03 M7.2 Hualien
07:58 EQ -> 08:11 Tsunami W1 -> 10:00 W2 -> 11:10 Cancel

#### Case: 2025/07/30 M8.8 Kamchatka
07:25 EQ -> 07:38 Info -> 09:12 Alert -> 11:40 Warning -> 17:10 Cancel

---

## Distribution Network

- Direct to **>4500 units**: 4100 schools, highways, railways, NCDR

### AI for Intensity
- **3 sec P-wave** -> ~**15 sec leading time**

---

## Conclusion
1. Continuous EEW performance improvement
2. Strengthen public-private cooperation
3. From 102 sec to 7 sec - constant evolution
